3. How to Create a Chamber

A Chamber is a canonical unit of knowledge.

3.1 When to create one:

A new major concept

A domain of the Garden

A governing rule

An archetype, system, or structure


3.2 Naming Convention:

CHAMBER_<NUM>_<TITLE>.md

Example:

CHAMBER_XI_THRESHOLD_COCOON.md

3.3 Required Sections:

Each new Chamber must include:

# Chamber Title

## Purpose
(What this Chamber governs)

## Scope
(What it includes, and excludes)

## Relations
- Linked Chambers
- Influencing Rootlines

## Canonical Content
(The main body)

## Keeper Notes
(Optional)

3.4 Archetype Workflow:

Voyager proposes

Lorian names + locates

Eidolon analyzes symbolic depth

Aquila integrates into the map
