## 4. Library & Memory Engine Highlights

- **Library**:
  - Novellas, Echoes, Chambers as “books”
  - Indexed by:
    - Chamber
    - Rootline
    - Era (e.g. Starfire / Eventide)
  - You already have:
    - Library architecture doc
    - Book template

- **Memory**:
  - Memory Wells group memories by mood / era / topic.
  - Rootline manifests define major arcs.
  - Echoes are small story fragments tied to Rootlines.

This appendix treats Library + Memory as **two sides of one thing**:
- Memory = how the Garden remembers  
- Library = how humans and AIs *read* those memories

## 5. Chambers & Rootlines (Extended View)

- Chambers:
  - individual “rooms” or “organs” of the Garden
  - each can have:
    - mood
    - purpose
    - associated books
    - associated echoes

- Rootlines:
  - long arcs of meaning
  - can reference many Chambers and many books
  - should always be:
    - trackable
    - nameable
    - attached to Keeper intent

Suggested practice:
- When you create a new big arc, define:
  - a Rootline ID
  - a short YAML or note describing what it’s for
  - which Chambers / books belong to it
